# Sandbox Doc Bot — 이 기능은 어떻게 만들어졌나

**대상**: 클로드를 처음 써보는 팀원
**목적**: "업무 자동화를 클로드에게 어떻게 요청하면 되는지" 실제 제작 과정으로 보여주기

---

## 발표자용 라이브 시연 스크립트

> 이 섹션은 발표자가 팀원 앞에서 클로드를 실시간으로 보여줄 때 쓰는 대본입니다.
> 화면 공유 상태에서 클로드 창을 띄워두고, 아래 순서대로 진행합니다.

---

### 시연 전 사전 준비 (발표 시작 전)

**필요한 도구 설명 (팀원에게 먼저 말하기):**

> "오늘 제가 쓸 도구는 세 가지입니다.
> 첫째, **클로드 코드(Claude Code)** — AI한테 말을 걸면서 코드를 짜게 시키는 도구입니다.
> 둘째, **파이썬(Python)** — 컴퓨터가 이해하는 언어인데, 오늘은 직접 코딩 안 해요. 클로드가 다 짭니다.
> 셋째, **MCP** — 클로드가 Slack, Notion, 캘린더 같은 외부 서비스를 직접 연결해서 쓸 수 있게 해주는 연결 플러그인입니다.
> 쉽게 말하면 클로드한테 '팔'을 붙여주는 거예요. 말만 하면 클로드가 실제로 Slack에 메시지를 보내거나 파일을 읽어옵니다."

---

### 시연 1: "클로드가 뭘 할 수 있는지" 보여주기 (3분)

화면에 클로드 코드 창 띄우고, 아래 입력:

```
헬프데스크 Slack 채널에 "사업자등록증 받을 수 있을까요?" 라는
메시지가 오면 어떻게 자동 답변을 만들 수 있을지 설명해줘.
코드 바로 짜지 말고, 어떤 방식이 있는지 옵션만 먼저 알려줘.
```

> **팀원에게 설명:**
> "코드 짜기 전에 방향을 먼저 물어봤습니다.
> 클로드한테 바로 '만들어줘'라고 하면 알아서 결정해버리는데,
> 이렇게 물어보면 내가 선택할 수 있어요."

---

### 시연 2: "핵심 로직 짜기" 보여주기 (5분)

```
방식 B (키워드 매칭)로 가자.
텍스트 입력받으면 문서 요청인지 판단하는 함수 하나만 만들어줘.
Slack 연동은 아직 말고, 그냥 파이썬 함수로만.
```

> **팀원에게 설명:**
> "전체를 한 번에 만들려 하지 않습니다.
> '이것만 먼저'라고 범위를 줄이면 클로드가 딱 그만큼만 만들어요.
> 파이썬은 클로드가 다 짭니다. 저는 '뭘 만들지'만 결정했어요."

코드가 나오면:
```
이걸 터미널에서 테스트해볼게. 어떻게 확인하면 돼?
```

> **팀원에게 설명:**
> "이렇게 묻기만 하면 클로드가 테스트 방법도 알려줍니다."

---

### 시연 3: "조건 추가하기" 보여주기 (3분)

```
잘 되네. 그런데 인감 관련 요청은 제외해야 해.
그건 직접 처리해야 하거든. 필터 추가해줘.
```

> **팀원에게 설명:**
> "처음부터 다 생각해서 줄 필요 없습니다.
> 써보다가 '이것도 필요하다' 싶으면 그냥 말하면 됩니다.
> 클로드가 기존 코드에 추가해줘요."

---

### 시연 4: "문서 목록을 코드 밖으로 빼기" 보여주기 (2분)

```
문서 목록은 내가 나중에 직접 추가할 수 있어야 해.
코드 안에 넣지 말고 별도 파일로 만들어줘.
```

> **팀원에게 설명:**
> "이게 핵심입니다. '내가 나중에 고칠 수 있어야 해'라는 조건만 주면
> 클로드가 구조를 거기에 맞게 설계합니다.
> 이 JSON 파일은 코딩 몰라도 메모장으로 수정할 수 있어요."

> 1. 뭘 만들지 설명했고
> 2. 클로드가 제안한 방향 중 하나를 골랐고

|---------|------|

## 배경 — 왜 만들었나

GA 헬프데스크 채널에는 매주 비슷한 문의가 반복됩니다.

> "사업자등록증 받을 수 있을까요?"
> "회사소개서 어디서 구하나요?"
> "재직증명서 발급 부탁드려요"

담당자가 매번 Notion 자료실 링크를 복사해서 보내는 게 일이었습니다.
이걸 자동화하고 싶었습니다.

---

## 1단계: 클로드에게 문제를 설명한다 (기술 용어 없이)

처음부터 "파이썬으로 Slack API 연동해줘" 식으로 말하면 안 됩니다.
**클로드에게도 신입한테 설명하듯이** 상황부터 설명했습니다.

**실제로 사용한 프롬프트:**
```
GA 헬프데스크 Slack 채널에서 사람들이 사업자등록증, 회사소개서
같은 문서를 달라는 요청이 자꾸 와.

지금은 내가 직접 노션 자료실 링크를 찾아서 붙여넣고 있는데,
이게 자동으로 됐으면 좋겠어.

어떻게 하면 될까?
```

**클로드가 한 것:**
- "키워드 감지 → 자동 답변" 방식 제안
- 필요한 것 목록 정리 (Slack 앱, Notion 연동, 문서 목록 정의)
- 어떤 순서로 만들지 물어봄

> **포인트**: 기술 설명 없이 "업무 상황 + 원하는 결과"만 말해도 됩니다.
> 클로드가 어떻게 구현할지 스스로 설계합니다.

---

## 2단계: 설계를 함께 결정한다

클로드가 방향을 제안하면, 내가 선택합니다.

**클로드의 제안:**
```
두 가지 방식이 있습니다.

방식 A — AI가 문의 내용을 판단해서 답변
  장점: 자연어로 된 문의도 잡아낼 수 있음
  단점: API 비용 발생, 오답 가능성

방식 B — 키워드 목록으로 매칭
  장점: 빠르고 비용 없음, 예측 가능
  단점: 키워드 목록에 없는 표현은 못 잡음

어떤 방식으로 하시겠어요?
```

**내가 선택한 것:** 방식 B (키워드 매칭)
- 문서 요청은 표현이 어느 정도 정해져 있으니까
- 비용 없이 빠르게 동작하는 게 우선

> **포인트**: 클로드가 옵션을 제시하면 내가 방향을 결정합니다.
> "알아서 다 해줘"보다 이렇게 주요 결정에는 같이 참여하는 게 좋습니다.

---

## 3단계: 핵심 로직부터 만든다

전체를 한번에 만들려 하지 않고, **가장 핵심이 되는 것부터** 시작했습니다.

**프롬프트:**
```
문서 요청 감지 로직부터 만들어줘.
Slack이나 Notion 연동 없이, 그냥 텍스트 넣으면
문서 요청인지 아닌지 판단하는 함수 하나만.
```

**결과물 (`doc_request_agent.py` 핵심 부분):**
```python
def detect_document_request(text: str) -> dict | None:
    # 1단계: 특정 문서 키워드 매칭
    for doc in documents:
        for alias in doc["aliases"]:
            if alias in text:
                return {"name": doc["name"], "notion_url": doc["notion_url"]}

    # 2단계: 일반 서류 요청인지 확인
    has_doc_keyword = any(kw in text for kw in ["서류", "자료", "문서", "증명서"])
    has_request_verb = any(v in text for v in ["받을", "주세요", "보내", "필요"])

    if has_doc_keyword and has_request_verb:
        return {"name": "자료실", "notion_url": LIBRARY_URL}

    return None  # 문서 요청 아님
```

**테스트 (Slack 연동 전):**
```python
# 터미널에서 바로 확인
result = detect_document_request("사업자등록증 받을 수 있을까요?")
print(result)
# → {"name": "사업자등록증", "notion_url": "https://notion.so/..."}

result = detect_document_request("오늘 날씨 어때요?")
print(result)
# → None
```

> **포인트**: 처음부터 완성품을 만들려 하지 않습니다.
> "이것만 먼저" → 확인 → "다음 것 추가" 순서로 진행합니다.

---

## 4단계: 문서 목록을 데이터로 분리한다

클로드가 코드 안에 문서 목록을 직접 넣으려 하자, 이렇게 요청했습니다.

**프롬프트:**
```
문서 목록은 나중에 내가 직접 추가할 수 있어야 해.
코드 안에 하드코딩하지 말고 별도 파일로 분리해줘.
```

**결과물 (`knowledge/documents.json`):**
```json
[
  {
    "name": "사업자등록증",
    "aliases": ["사업자등록증", "사업자 등록증", "사업자번호"],
    "notion_url": "https://www.notion.so/sandboxinc/...",
    "notion_page_id": "...",
    "local_file": "사업자등록증.pdf",
    "description": "법인 사업자등록증 사본"
  }
]
```

이렇게 하면 앞으로 문서를 추가할 때 코드를 건드릴 필요 없이
이 JSON 파일만 수정하면 됩니다.

> **포인트**: "나중에 내가 직접 고칠 수 있어야 해" 라는 조건을 주면
> 클로드가 구조 자체를 거기에 맞게 설계합니다.

## 5단계: Slack 연동을 붙인다

로직이 검증된 뒤에야 Slack 연동을 추가했습니다.

**프롬프트:**
```
감지 로직은 잘 되는 거 확인했어.
이제 이걸 Slack 헬프데스크 채널에 연결해줘.
새 메시지 올라오면 감지해서, 문서 요청이면 스레드에 바로 답변하도록.
```

**추가 조건 (한 번에 다 말하지 않고, 테스트하면서 추가):**
- "근무시간 외엔 동작 안 하게 해줘" → `is_work_hours()` 추가
- "인감 관련 요청은 제외해줘, 그건 직접 처리해야 해" → `EXCLUDE_KEYWORDS` 추가
- "바로 답변하지 말고 팀 채널에 먼저 보여줘서 내가 승인하게 해줘" → TEST_MODE + 버튼 카드 추가

> **포인트**: 처음부터 모든 요구사항을 정리해서 줄 필요 없습니다.
> 써보다가 "이것도 필요하다" 싶으면 그때 추가 요청합니다.

## 6단계: Notion 파일 자동 다운로드를 추가한다

처음엔 Notion 링크만 안내했는데, 파일을 바로 전송하면 더 좋겠다 싶었습니다.

**프롬프트:**
```
Notion 링크 안내만 하는 게 아니라
파일을 Slack에 직접 첨부해서 보내줄 수 있어?
Notion에 올라간 PDF 파일을 가져와서.
```

**결과물 (`refresh_docs.py`):**
Notion API로 파일을 다운로드 → `knowledge/files/` 폴더에 저장
봇 실행 중 문서 요청 오면 → 로컬 파일 Slack에 업로드

**사용 방법:**
```bash
python refresh_docs.py  # Notion 파일 로컬 저장
# 이후 봇이 자동으로 파일 첨부 전송
```

## 전체 흐름 정리

```
(처음)  문제 상황 설명
         ↓
(설계)  클로드가 방향 제안 → 내가 선택
         ↓
(구현)  핵심 로직 먼저 → 테스트 → 연동 → 기능 추가
         ↓
(운영)  쓰다가 필요한 것 추가 요청
```

## 클로드에게 일 시키는 3가지 원칙

**1. 기술 용어 없이, 업무 상황으로 말한다**
- ❌ "Slack Bolt로 소켓모드 이벤트 핸들러 만들어줘"
- ✅ "헬프데스크 채널에 메시지 오면 자동으로 답변하게 해줘"

**2. 한번에 다 안 줘도 된다**
- 처음엔 핵심만 → 쓰다가 필요한 것 그때그때 추가
- 완벽한 요구사항 정리는 시간 낭비

**3. 중요한 결정은 내가 한다**
- 클로드가 옵션 제시 → 내가 선택
- "알아서 해줘"보다 이쪽이 결과가 더 내 의도에 맞음

## 운영 가이드 (실무용)

### 새 문서 추가
`knowledge/documents.json`에 항목 추가 후 `python refresh_docs.py` 실행.

```json
{
  "name": "재직증명서",
  "aliases": ["재직증명", "재직 증명서", "재직확인서"],
  "notion_url": "https://www.notion.so/sandboxinc/...",
  "notion_page_id": "페이지-ID",
  "local_file": "재직증명서.pdf",
  "description": "인사팀 발급"
}
```

### 테스트 방법
Slack 팀 채널에서: `test: 사업자등록증 보내주세요`
→ 봇이 스레드로 결과 출력 (헬프데스크 실제 전송 없음)

### 프로덕션 전환
`.env`에서 `TEST_MODE=false` → 재시작
